# Product

## Register

product

## Users

Synarava staff, founders, editors, and merchandisers use the admin area to maintain a small luxury jewelry storefront. They work in focused sessions: adding products, shaping editorial collection pages, checking publish state, and managing account access.

## Product Purpose

Synarava is a couture jewelry storefront with a lightweight CMS. The admin exists to keep catalog, collection, and editorial content accurate, publishable, and safe to edit without engineering help.

## Brand Personality

Quiet, ceremonial, precise. The public brand is editorial and museum-like; the admin should translate that into a calm studio console rather than a decorative marketing surface.

## Anti-references

Avoid generic SaaS dashboards, neon terminal themes, over-decorated glassmorphism, weak low-contrast metadata, and noisy command-center language that makes ordinary content tasks feel technical.

## Design Principles

- Preserve the brand atmosphere while making every CMS task legible and calm.
- Make publishing state, draft risk, and locale scope visible before the user edits.
- Keep the interface dense enough for repeated work, but never cryptic.
- Prefer reusable admin primitives over one-off page styling.
- Treat destructive actions and authentication changes as high-friction, clearly explained workflows.

## Accessibility & Inclusion

Target WCAG AA contrast for admin text and controls. Support reduced motion, keyboard focus states, visible error messaging, and touch targets that remain usable on mobile and tablet.

# Synarava Jewelry — Мобильное приложение (iOS & Android)

Этот каталог содержит исходный код мобильного приложения для бренда **Synarava Jewelry**, построенного на базе технологии **Capacitor**. 

Приложение работает как ультрабыстрый **Live Wrapper** (нативная обертка) для вашего существующего Next.js веб-сайта, обеспечивая мгновенную синхронизацию базы данных, панели администратора (`/admin`) и всех функций интернет-магазина.

---

## 🛠️ Предварительные требования

Для сборки и запуска мобильных приложений на вашем Mac вам понадобятся:
1. **Node.js** (версии 18 или новее)
2. **Xcode** (для сборки iOS-приложения, можно скачать из Mac App Store)
3. **Cocoapods** (для установки зависимостей iOS):
   ```bash
   sudo gem install cocoapods
   # или через Homebrew:
   brew install cocoapods
   ```
4. **Android Studio** (для сборки Android-приложения) c установленным Android SDK и настроенным эмулятором.

## 🚀 Быстрый старт (Локальное тестирование)

### Шаг 1. Запуск веб-сайта
Убедитесь, что ваш основной Next.js проект запущен локально (в корневом каталоге `synarava-jewelry`):
```bash
# В корне synarava-jewelry
pnpm dev
# (или npm run dev)
```
Веб-сайт должен быть доступен по адресу `http://localhost:3000`.

### Шаг 2. Установка зависимостей мобильного проекта
Откройте новый терминал и перейдите в папку `mobile`:
```bash
cd mobile
npm install
```

### Шаг 3. Инициализация и синхронизация Capacitor
Синхронизируйте конфигурацию Capacitor и загрузите нативные платформы:
```bash
# Добавление нативных платформ (делается один раз)
npx cap add ios
npx cap add android

# Синхронизация файлов конфигурации и плагинов
npx cap sync
```

### Шаг 4. Запуск в симуляторах

#### Запуск на iOS (через Xcode):
Выполните команду для открытия проекта в Xcode:
```bash
npx cap open ios
```
В открывшемся Xcode выберите желаемое устройство (например, *iPhone 15*) в верхней панели и нажмите кнопку **Run** (иконка треугольника Play).

#### Запуск на Android (через Android Studio):
Выполните команду для открытия проекта в Android Studio:
```bash
npx cap open android
```
В открывшейся Android Studio подождите окончания индексации Gradle, выберите виртуальное устройство (эмулятор) и нажмите кнопку **Run** (зеленый треугольник).

## ⚙️ Конфигурация для Продакшна

В файле `mobile/capacitor.config.ts` настроена конфигурация для локальной разработки:
```typescript
server: {
  url: 'http://localhost:3000', // Локальный сервер Next.js
  cleartext: true
}
```

Перед отправкой приложения в App Store и Google Play измените эти настройки на ваш реальный рабочий домен:
```typescript
server: {
  url: 'https://synarava.com', // Замените на ваш рабочий домен
  cleartext: false // Отключает небезопасное HTTP-соединение
}
```
После изменения конфигурации выполните `npx cap sync` в папке `mobile`.

## 📱 Адаптация веб-интерфейса под мобильный WebView

Поскольку мобильное приложение отображает ваш веб-сайт, вы можете легко управлять элементами интерфейса, которые не нужны в приложении (например, подвал сайта или ссылки на скачивание приложения).

### Метод 1: Скрытие элементов через CSS
Capacitor автоматически добавляет CSS-класс `.capacitor-app` на элемент `<html>` при запуске внутри приложения. Вы можете использовать этот класс в `app/globals.css`:
```css
/* Скрываем футер сайта только внутри мобильного приложения */
.capacitor-app footer {
  display: none !important;
}

/* Делаем шапку более компактной в мобильном приложении */
.capacitor-app .artifact-nav {
  padding-top: 1rem;
  padding-bottom: 1rem;
}
```

### Метод 2: Проверка в JavaScript / React
Вы можете проверять, запущено ли приложение в нативной оболочке, с помощью проверки платформы Capacitor:
```typescript
import { Capacitor } from '@capacitor/core';

if (Capacitor.isNativePlatform()) {
  console.log("Запущено на iOS или Android в нативном контейнере!");
} else {
  console.log("Запущено в обычном браузере на десктопе или мобильном");
}
```
